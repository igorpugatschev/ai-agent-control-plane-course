# AI Agent Control Plane Engineering

Курс по проектированию и развитию управляющего слоя для AI-агентов в инженерных процессах: разработке, анализе кода, QA, SDET, документации, ревью и автоматизации рабочих контуров.

Это не курс по “хорошим промптам” в узком смысле. Промпты здесь являются только одним из элементов системы. Главный фокус курса - научиться строить управляемую AI-agent среду поверх готовых LLM: ChatGPT, DeepSeek, Qwen, локальных моделей и других инструментов.

## Принцип источников

Основа курса - официальная документация инструментов, API, SDK, CLI и платформ, с которыми работает студент.

При этом курс должен быть самодостаточным: обязательная теория, терминология, задания и критерии готовности находятся внутри репозитория. Ссылки на официальные первоисточники приветствуются, но они служат для проверки и углубления, а не заменяют учебный материал.

Весь обязательный материал курса должен быть на русском языке. Английские термины, названия инструментов, API, SDK, CLI, параметры, команды и фрагменты кода допускаются в оригинале, но должны сопровождаться понятным русским объяснением.

Подробно: [политика источников и самодостаточности](docs/source-policy.md).

Канонический список проверенных первоисточников: [каталог достоверных источников](docs/source-catalog.md).

## Миссия курса

Подготовить инженера, который умеет не просто пользоваться нейросетями, а проектировать рабочий AI-agent control plane:

- описывать роли агентов и границы их ответственности;
- создавать правила, скиллы, инструкции и маршруты выполнения задач;
- подключать контекст проекта, документацию, исходный код и внешние источники;
- строить проверки качества, stop-gates и review-gates;
- применять агентные процессы в разработке, QA и SDET;
- превращать разовые AI-чаты в воспроизводимые инженерные workflow.

Подробно: [миссия и целевая роль](docs/course-mission.md).

## Целевая роль

Рабочее название роли:

**AI Agent Control Plane Engineer**

Практический профиль выпускника:

**инженер, который строит управляющий слой для AI-агентов в разработке, QA и SDET-процессах.**

Такой специалист находится на пересечении нескольких областей:

- software engineering;
- QA automation / SDET;
- LLM engineering;
- agentic workflows;
- developer tooling;
- process automation;
- technical documentation;
- quality gates and review systems.

## Чем это отличается от Prompt Engineering

Prompt Engineer пишет сильные инструкции для модели.

AI Agent Control Plane Engineer проектирует систему вокруг модели:

- какие агенты существуют;
- какие задачи они берут;
- какие инструменты используют;
- как получают контекст;
- как передают работу друг другу;
- где обязаны остановиться;
- как проверяется качество результата;
- как этот процесс повторяется в команде.

## Учебный маршрут

Стартовая версия курса строится вокруг семи смысловых блоков:

1. База: что такое AI-agent control plane и зачем он нужен.
2. Контекст: как агент получает проектную память, документацию и исходники.
3. Роли: как проектировать агентов, скиллы, правила и зоны ответственности.
4. Workflow: как собирать повторяемые процессы для разработки, QA и SDET.
5. Gates: как проверять качество, управлять рисками и останавливать слабые результаты.
6. Safety and review: как задавать stop-gates, review-gates и human approval.
7. Capstone: как собрать собственный control plane для реального инженерного процесса.

Подробно: [roadmap курса](curriculum/roadmap.md).

## Первый результат обучения

После первого этапа студент должен уметь:

- объяснить разницу между prompt, agent, skill, workflow и control plane;
- описать целевую инженерную задачу как агентный процесс;
- выделить роли агентов и правила их взаимодействия;
- определить источники контекста и проверки качества;
- оформить минимальный control-plane blueprint для своего проекта.
