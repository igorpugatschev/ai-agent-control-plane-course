# Модуль 2. Контекст и доказательства

Модуль учит собирать только те данные, на которых можно безопасно принять инженерное решение. Сквозной пример - намеренно устаревшая инструкция в `projects/training-task-app/scenarios/stale-documentation.md`: она предлагает завершать задачу по названию и получать `closed`, хотя текущий контракт использует целочисленный `task_id` и статус `done`.

К концу модуля вы не меняете код и не переписываете конфликтующую документацию по догадке. Вы создаете три связанных артефакта:

1. [Источники, факты и свежесть](lesson-04-source-hierarchy.md) - `artifacts/module-02/source-register.md`.
2. [Минимальный context packet](lesson-05-context-packet.md) - `artifacts/module-02/context-packet.md`.
3. [Карта контекста и evidence gate](lesson-06-context-map-evidence-gate.md) - `artifacts/module-02/context-map-evidence-gate.md`.
4. [Checkpoint 2](checkpoint.md) - проверка трех артефактов на конфликте stale-документации.

## Обязательный локальный маршрут

Сначала прочитайте локальные входные данные: `requirements.md`, `api/openapi.yaml`, `scenarios/stale-documentation.md` и `tests/test_service.py` учебного стенда. Уроки содержат подготовленные ответы, поэтому обязательный путь не требует платного API-ключа, облачного аккаунта или внешней документации. Необязательный живой агент может подготовить черновик, но его текст не становится evidence без проверки по локальным файлам и командам.

Во всех уроках разделяйте источник, `Source owner` и `Authority owner`. `Source owner` отвечает за происхождение или сопровождение конкретного источника; `Authority owner` вправе принять решение по контракту. Это не взаимозаменяемые роли. Не назначайте владельца по догадке: `unknown` - явный пробел с эскалацией к `Authority owner` и `STOP` до документирования. При конфликте нет права выбрать более удобную формулировку. Правильный результат - `STOP`, журнал расхождения и запрос решения Product owner.

## Готовность модуля

Модуль завершен, если:

- у каждого существенного вывода есть repo-relative источник, `Source owner` либо документированное `unknown` с эскалацией, `Authority owner`, дата проверки и confidence;
- минимальный пакет не включает весь репозиторий и отделяет доверенные инструкции от недоверенных данных;
- карта контекста использует структуру `templates/context-map.md` и показывает исключения, свежесть и пробелы;
- evidence gate требует воспроизводимой команды или проверяемого файла, а не фразы «агент подтвердил»;
- конфликт между stale-документацией и текущим контрактом приводит к STOP до любого неподтвержденного изменения.
