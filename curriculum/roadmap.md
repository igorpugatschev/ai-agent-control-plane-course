# Roadmap курса

Это стартовая карта курса. Она фиксирует направление, но не является окончательной программой.

## Модуль 1. Что такое AI Agent Control Plane

Цель: отделить разовое использование LLM от управляемого инженерного процесса.

Темы:

- LLM, prompt, agent, skill, workflow, control plane;
- почему “просто хороший промпт” не масштабируется;
- где AI-агенты помогают в разработке, QA и SDET;
- риски агентной автоматизации;
- роль человека как владельца процесса.

Результат:

- студент описывает свой первый control-plane blueprint на 1 страницу.

## Модуль 2. Контекст и проектная память

Цель: научиться давать агентам проверяемый контекст.

Темы:

- источники контекста;
- документация и исходный код;
- live-состояние проекта;
- внешние справочники и документация;
- stale context и способы защиты от него;
- evidence-based ответы.

Результат:

- студент описывает карту источников контекста для учебного проекта.

## Модуль 3. Роли, правила и скиллы агентов

Цель: научиться проектировать поведение агентов как инженерный контракт.

Темы:

- роль агента;
- зона ответственности;
- входные данные;
- выходные артефакты;
- ограничения;
- правила остановки;
- handoff другому агенту или человеку.

Результат:

- студент создает первые agent-role и skill templates.

## Модуль 4. Workflow для разработки

Цель: собрать повторяемые агентные процессы для software engineering.

Темы:

- анализ задачи;
- планирование изменений;
- работа с кодовой базой;
- review;
- тестирование;
- documentation update;
- commit / PR handoff.

Результат:

- студент описывает workflow для небольшой инженерной задачи.

## Модуль 5. Workflow для QA и SDET

Цель: применить control-plane подход к качеству продукта.

Темы:

- анализ требований;
- тест-дизайн;
- API/UI automation;
- regression planning;
- defect investigation;
- flaky triage;
- traceability;
- release gates.

Результат:

- студент проектирует QA/SDET agent workflow с критериями приемки.

## Модуль 6. Gates, review и безопасность процесса

Цель: научиться не только запускать агентов, но и контролировать качество их работы.

Темы:

- verification gates;
- stop-gates;
- confidence levels;
- hallucination control;
- источники истины;
- human approval;
- журналирование решений;
- защита от prompt injection.

Результат:

- студент добавляет review-gates в свой учебный control plane.

## Модуль 7. Capstone project

Цель: собрать полноценный управляющий слой для реального или приближенного к реальному инженерного процесса.

Варианты capstone:

- control plane для code review;
- control plane для QA regression;
- control plane для SDET automation;
- control plane для документации проекта;
- control plane для анализа дефектов;
- control plane для onboarding в кодовую базу.

Итог:

- README проекта;
- карта ролей агентов;
- workflow-документация;
- templates;
- gates;
- пример запуска;
- отчет о рисках и ограничениях.
