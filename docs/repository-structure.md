# Структура репозитория

Этот репозиторий устроен как учебный control plane: здесь хранятся не только уроки, но и правила, шаблоны, роли агентов и практические проекты.

## Основные директории

```text
.
├── README.md
├── docs/
├── curriculum/
├── agents/
├── projects/
└── templates/
```

## `docs/`

Концептуальные документы курса:

- миссия;
- целевая роль;
- политика источников и самодостаточности;
- терминология;
- архитектурные принципы;
- правила качества;
- методология работы.

Эти документы отвечают на вопрос: **зачем и по каким принципам устроен курс**.

Ключевое правило: официальная документация является первоисточником для автора курса, но обязательная теория должна быть изложена внутри репозитория.

## `curriculum/`

Учебный маршрут:

- модули;
- уроки;
- результаты обучения;
- checkpoints;
- практические задания;
- критерии готовности.

Эта директория отвечает на вопрос: **в какой последовательности учиться**.

## `agents/`

Описание учебных и рабочих агентов:

- роли;
- зоны ответственности;
- правила поведения;
- входы и выходы;
- handoff между агентами;
- review и stop conditions.

Эта директория отвечает на вопрос: **какие агенты участвуют в процессе и как они должны работать**.

## `projects/`

Практические проекты курса.

Здесь должны появляться задания, где студент собирает собственные управляющие слои:

- для разработки;
- для QA;
- для SDET;
- для документации;
- для code review;
- для regression planning;
- для анализа дефектов.

Эта директория отвечает на вопрос: **что студент реально строит руками**.

## `templates/`

Переиспользуемые шаблоны:

- шаблон роли агента;
- шаблон skill-инструкции;
- шаблон workflow;
- шаблон review-gate;
- шаблон отчета;
- шаблон итогового blueprint.

Эта директория отвечает на вопрос: **какие заготовки можно использовать повторно**.

## Принцип развития

Каждый новый учебный материал должен отвечать хотя бы на один из вопросов:

- какую инженерную проблему он решает;
- какую часть control plane он усиливает;
- какой навык студент тренирует;
- как проверить, что навык освоен;
- как этот материал можно применить в реальном проекте.
