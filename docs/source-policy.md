# Политика источников и самодостаточности

Курс строится по принципу **official-docs-first**: все ключевые понятия, практики и ограничения должны выводиться из официальной документации продуктов, API, SDK, CLI и инструментов, с которыми работает студент.

При этом курс остается **самодостаточным**: студент должен иметь возможность пройти основной маршрут после клонирования репозитория, без обязательного поиска внешней литературы, статей, видео, книг или чужих учебников.

## Роль официальной документации

Официальная документация является главным первоисточником для автора курса.

К ней относятся:

- документация OpenAI, Codex, ChatGPT, API, SDK и Agents-инструментов;
- официальная документация провайдеров моделей и LLM-инструментов;
- официальная документация языков, фреймворков, библиотек и CLI;
- официальные changelog, release notes, migration guides и API reference;
- официальные репозитории, если они являются каноническим источником поведения инструмента.

Если в курсе объясняется поведение инструмента, ограничение API, формат конфигурации, модель безопасности, workflow или рекомендуемая практика, автор курса должен сверять это с официальным первоисточником.

Канонический список проверенных источников ведется в [каталоге достоверных источников](source-catalog.md). Каталог фиксирует уровни доверия, роль источника в курсе и ограничения его применения.

## Роль материалов внутри курса

Основная теория должна находиться внутри репозитория курса.

Студент не должен быть вынужден:

- искать объяснение темы в другой книге;
- смотреть внешние видео, чтобы понять обязательный материал;
- читать длинную внешнюю документацию вместо урока;
- собирать базовую теорию из статей, блогов или чужих курсов;
- угадывать, какая версия внешнего материала соответствует уроку.

Каждый обязательный урок должен давать внутри курса:

- объяснение ключевых понятий;
- минимальную терминологию;
- практический контекст;
- пошаговое задание;
- критерии готовности;
- типичные ошибки и ограничения;
- ссылки на официальные первоисточники для проверки и углубления.

## Язык курса

Весь обязательный материал курса должен быть на русском языке.

Это относится к:

- теории;
- объяснениям терминов;
- учебным заданиям;
- контрольным вопросам;
- критериям готовности;
- шаблонам;
- комментариям к внешним источникам;
- разбору типичных ошибок.

Английский язык допустим там, где он является частью технической реальности:

- названия продуктов, моделей, библиотек и фреймворков;
- имена API, SDK, CLI, протоколов и форматов;
- команды терминала;
- параметры конфигурации;
- имена файлов, классов, функций и переменных;
- короткие цитаты или заголовки из официальной документации.

Если английский термин важен для курса, рядом должно быть русское объяснение. Студент не должен понимать материал только потому, что уже свободно читает англоязычную документацию.

Официальные курсы и учебные репозитории на английском языке можно использовать как источники для автора курса, но обязательная теория должна быть адаптирована и изложена в репозитории на русском языке.

## Как использовать ссылки

Ссылки на официальную документацию приветствуются, но они не заменяют учебный материал.

Правильный формат:

- сначала дать самодостаточное объяснение в уроке;
- затем указать официальный источник;
- кратко объяснить, что именно студент может проверить по ссылке;
- при необходимости указать дату проверки или версию инструмента.

Неправильный формат:

- "прочитайте документацию и разберитесь";
- "изучите статью/видео, а потом выполните задание";
- "вот список ссылок вместо теории";
- "материал будет понятен только после внешнего курса".

## Вторичные источники

Блоги, статьи, видео, личный опыт, обсуждения и книги других авторов можно использовать только как вторичные материалы.

Они допустимы для:

- примеров;
- альтернативных объяснений;
- исторического контекста;
- авторских заметок;
- необязательного углубления.

Они не должны быть источником истины для обязательной программы курса.

## Правило обновления

Если официальная документация изменилась, курс должен обновляться через controlled review:

1. найти изменившийся официальный источник;
2. понять, какие уроки, шаблоны или проекты затронуты;
3. обновить теорию внутри курса;
4. обновить задания и критерии готовности;
5. оставить ссылку на официальный источник как подтверждение;
6. проверить, что основной маршрут по-прежнему можно пройти без внешнего поиска.
