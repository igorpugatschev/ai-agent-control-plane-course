# Учебный стенд задач

## Цель

Этот проект даёт учебный, воспроизводимый контур для работы с задачами: создать задачу, получить список задач и завершить задачу. Он используется как общий пример в модулях 2, 4, 5 и 6 курса.

## Контекст

Стенд реализован как небольшая библиотека Python без HTTP-сервера, базы данных и внешних сервисов. `TaskService` хранит задачи в памяти одного экземпляра сервиса. Это позволяет разбирать границы задачи, контракт, дефект, нестабильный запуск и конфликт документации на одном детерминированном примере.

## Требования

- Python 3.11 или новее;
- стандартная библиотека Python для исходного кода;
- `pytest` для тестов;
- доступ к сети и постоянное хранилище не требуются.

Подробные требования находятся в [`requirements.md`](requirements.md), формальный контракт в [`api/openapi.yaml`](api/openapi.yaml).

## Запуск

Из корня репозитория выполните:

```bash
PYTHONPATH=projects/training-task-app/src python3 -m pytest projects/training-task-app/tests -q
```

Ожидаемый результат: одиннадцать пройденных тестов.

Для интерактивной проверки:

```python
from task_app import TaskService

service = TaskService()
task = service.create_task("Проверить контракт")
print(service.list_tasks())
print(service.complete_task(task.id))
```

## Контракт

`TaskService.create_task(title)` обрезает внешние пробелы, отклоняет пустое название и не позволяет создать вторую активную задачу с тем же названием без учёта регистра. После завершения задачи её название можно использовать снова. Идентификаторы начинаются с `1` и выдаются последовательно.

`TaskService.list_tasks()` возвращает задачи по возрастанию идентификатора. `TaskService.complete_task(task_id)` переводит существующую задачу в статус `DONE`; для неизвестного идентификатора поднимается `KeyError` с сообщением `Задача {id} не найдена`.

Публичные элементы пакета: `Task`, `TaskStatus` и `TaskService`.

## Учебные сценарии

- [`scenarios/ambiguous-change.md`](scenarios/ambiguous-change.md) показывает запрос, которому не хватает границ и критериев готовности;
- [`scenarios/defect-report.md`](scenarios/defect-report.md) содержит воспроизводимый дефект с ожидаемым и фактическим результатом;
- [`scenarios/flaky-run.log`](scenarios/flaky-run.log) хранит историю повторяющегося нестабильного запуска;
- [`scenarios/stale-documentation.md`](scenarios/stale-documentation.md) показывает расхождение старой документации с текущим требованием.

Сценарии являются входными данными для упражнений. Они не импортируются тестовым набором и не меняют состояние стенда.
