# Устаревшая документация

## Фрагмент старой инструкции

> Для завершения задачи вызовите `complete_task` и передайте название задачи строкой. Сервис найдёт задачу по названию и вернёт статус `closed`.

## Текущее требование

Актуальный контракт требует передавать целочисленный `task_id`. Сервис меняет статус существующей задачи с `open` на `done`; статуса `closed` в модели нет. Для неизвестного идентификатора используется `KeyError` с сообщением `Задача {task_id} не найдена`.

## Проверка расхождения

1. Сверить текст требования с `projects/training-task-app/requirements.md`.
2. Сверить формальный HTTP-контракт с `projects/training-task-app/api/openapi.yaml`.
3. Запустить стабильные тесты из README.
4. Удалить старое описание или пометить его как исторический сценарий, не выдавая его за рабочий API.

Таким образом, этот файл описывает намеренный конфликт для упражнения на актуализацию документации и не является инструкцией к использованию сервиса.
