# Шаблоны курса

Шаблоны помогают оформлять агентные процессы как проверяемые инженерные артефакты. Скопируйте нужный Markdown-файл в проект, заполните поля и сохраните краткий пример как ориентир для команды.

- [Blueprint плоскости управления](control-plane-blueprint.md): цель, роли, правила и evidence системы.
- [Карта контекста](context-map.md): источники, доверие и пробелы данных задачи.
- [Роль агента](agent-role.md): ответственность, права, запреты и handoff исполнителя.
- [Контракт навыка](skill-contract.md): повторяемая процедура и ее evidence.
- [Передача работы](handoff.md): состояние задачи для следующего владельца.
- [Workflow](workflow.md): trigger, шаги, ветвления, gates и recovery.
- [Review-gate](review-gate.md): независимая проверка перед переходом.
- [Stop-gate](stop-gate.md): безопасная остановка рискованного действия.
- [Журнал решений](decision-log.md): контекст, выбор и условие пересмотра.
- [Финальный отчет](final-report.md): итог, проверки и остаточные риски.

Определения терминов доступны в [глоссарии](../glossary/terms.md). Шаблоны самодостаточны; [каталог достоверных источников](../docs/source-catalog.md) служит только для проверки и углубления.

Следующее действие: возьмите форму, указанную в текущем уроке, и сохраните
именованный artifact в [starter control plane](../projects/starter-control-plane/README.md).
Полное отображение всех десяти форм в capstone находится в
[capstone contract](../projects/capstone.md); после заполнения artifact
проверьте его по соответствующему [checkpoint guide](../assessments/checkpoints/README.md).
